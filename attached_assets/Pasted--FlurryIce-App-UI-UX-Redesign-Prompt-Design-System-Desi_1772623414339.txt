# FlurryIce App — UI/UX Redesign Prompt & Design System

## 🎯 Design Direction: "Arctic Luxury POS"
**Aesthetic**: Refined minimalism meets premium retail — clean whites, deep navy, and crisp ice-blue accents. 
Inspired by premium ice cream brands: cold, fresh, premium, trustworthy.

---

## 🔴 Current UI Issues (from screenshots)

1. **Inconsistent header heights & button positions** — burger menu floats at different sizes across screens
2. **Mixed color themes** — yellow/cream on Dashboard, dark navy on Transfers, white on Products/Sales (3 different feels = 1 broken app)
3. **Inconsistent button sizes** — Quick Action cards vary in padding/icon size
4. **Typography weight chaos** — some labels bold, some not, no clear hierarchy
5. **Card styles differ per screen** — rounded corners, shadows, and padding not standardized
6. **Status badges inconsistent** — PAID badge style doesn't match filter chips
7. **Bottom tab bar icons misaligned** with labels on some screens
8. **Dark/Light mode toggle** exists only on Dashboard header

### Header Component — UNIFIED (`components/common/AppHeader.tsx`)

```typescript
/**
 * Single unified header used on ALL screens.
 * Props control which elements appear.
 */
interface AppHeaderProps {
  title?: string;                    // screen title (undefined = show logo)
  showLogo?: boolean;                // show Flurry logo instead of title
  showMenu?: boolean;                // show hamburger
  showBack?: boolean;                // show back arrow
  rightActions?: React.ReactNode;    // custom right-side icons
  dark?: boolean;                    // dark (navy) header vs white
}
```

**Header Rules:**
- Height: always **56px** (STATUS_BAR + 56)
- Left zone: 44×44 tap target (menu OR back)
- Center: title (semibold, 17px) OR logo (max 120px wide)
- Right zone: up to 2 icon buttons, each 44×44
- Dark header = white icons/text (used on Dashboard)
- White header = navy icons/text (used on Sales, Products, Transfers)

## 📐 Screen-by-Screen Redesign Specs

### 1. Dashboard (`app/(tabs)/index.tsx`)
- **Header**: Dark navy gradient, show logo centered, menu left, logout right
- **Greeting**: white, regular weight, 15px | name bold 20px
- **Revenue section**: large white number (38px hero), stays in gradient zone
- **Metric Cards**: 2×2 grid, white cards with shadow, icon in soft-tinted circle (24px icon)
- **Quick Actions**: horizontal scroll row, each card 80px wide, icon 28px in colored circle, label 12px centered below
- **Section titles**: 17px semibold, textPrimary

### 2. Sales (`app/(tabs)/sales.tsx`)
- **Header**: white header, title "Sales History" in navy, menu left, print+cloud right
- **Search bar**: inputBg, radius 12, 44px tall, magnifier icon left
- **Filter chips**: all=navy filled, others=ghost outline; same height 36px, all same font
- **Summary pills** (Payé/Impayé): colored background pills, right-aligned row
- **Sale cards**: unified Card component, PAID badge top-left with StatusBadge, amount in accent color

### 3. Products (`app/(tabs)/products/index.tsx`)
- **Header**: white, title "Products" navy, menu left, refresh+cart right (same size)
- **View toggle** (grid/list): ghost button group, 36px height each
- **Product cards (list)**: Card component — price bold navy left, stock badge (blue pill), name right, image placeholder circle
- In-stock items: full color; out-of-stock: muted (opacity 0.5)

### 4. Transfers (`app/(tabs)/transfers/index.tsx`)
- **Header**: white header (NOT dark), title "Stock Transfers", Scan QR button = primary button sm variant
- **Transfer cards**: Card component with white background (not dark navy!)
- Arrow between locations: accent color icon
- Item count + value: secondary text, same style as Sales

### 5. Sidebar/Menu
- Overlay from left, 75% screen width
- White background, navy header "Menu"
- Each menu item: 56px tall, icon (22px) + label (17px medium), navy icon, navy text
- Active item: accentSoft background, accent icon
- Divider between groups

## 💡 Key Principles

- **One source of truth**: every color, size, spacing comes from `constants/`
- **Never hardcode** hex colors or pixel values in screen files
- **Shared components** for anything used on 2+ screens
- **Consistent 44px tap targets** for all interactive elements (accessibility)
- **All headers = 56px** — no exceptions
- **All cards** use the Card component — no custom card styles inline
